# Выгрузка проекта на GitHub — пошаговая инструкция

---

## Часть 1 — Создание репозитория на GitHub

1. Зайди на [github.com](https://github.com) → войди в аккаунт

2. Нажми **"+"** (верхний правый угол) → **"New repository"**

3. Заполни:
   - **Repository name:** `secure_tunnel`
   - **Description:** вставь описание проекта (350 символов из предыдущего шага)
   - **Visibility:** выбери `Public` или `Private`
   - **НЕ ставь** галочки "Add README", "Add .gitignore", "Choose a license" — всё это уже есть в проекте

4. Нажми **"Create repository"**

5. GitHub покажет страницу с командами — **скопируй URL репозитория**, он понадобится в следующем шаге. Выглядит так:
   ```
   https://github.com/ТВО_ИМЯ/secure_tunnel.git
   ```

## Часть 2 — Первоначальная настройка Git (если не делал раньше)

Открой командную строку (`Win+R → cmd`) и выполни один раз:

```
git config --global user.name "Твоё Имя"
git config --global user.email "твой@email.com"
```

## Часть 3 — Инициализация и первый коммит

Открой командную строку **в папке проекта** (`C:\Users\79197\Documents\secure_tunnel`).

```
git init
git remote add origin https://github.com/ТВО_ИМЯ/secure_tunnel.git
git add .
git status
```

После `git status` **внимательно просмотри список файлов** — убедись что лишнего нет (см. Часть 4).

```
git commit -m "Initial commit: SecureTunnel v1.0.0"
git branch -M main
git push -u origin main
```

При первом push GitHub попросит войти — откроется браузер, авторизуйся через него.

## Часть 4 — Что НЕ должно попасть на GitHub

> Эти файлы уже исключены через `.gitignore` — но проверь вручную перед первым `git add .`

| Файл / Папка | Почему нельзя |
|---|---|
| `cert.pem` | Твой TLS-сертификат |
| `key.pem` | **Приватный ключ** — компрометирует весь трафик |
| `.env` | Содержит `AUTH_SECRET` и IP-адреса VPS |
| `secure_tunnel_keys/*.key` | Статические X25519-ключи узлов |
| `chrome_profile/` | Личные данные браузера (куки, история) |
| `dist/` | Собранный `.exe` — большой (18 МБ), распространяй через Releases |
| `build/` | Артефакты PyInstaller, не нужны в репозитории |
| `.venv/`, `venv/` | Виртуальное окружение Python — сотни МБ |

**Как проверить** — после `git add .` выполни:
```
git status
```
Все файлы выше должны быть в разделе "Untracked files" или отсутствовать. Если видишь их в "Changes to be committed" — немедленно убери:
```
git reset HEAD имя_файла
```

## Часть 5 — Последующие обновления

Когда изменил код и хочешь загрузить обновления:

```
git add .
git commit -m "Описание что изменил"
git push
```

## Часть 6 — Создание Release и публикация .exe

Release — это отдельная страница на GitHub, где пользователи скачивают готовый `.exe` одной кнопкой.

### Шаг 1 — Убедись что последние изменения загружены

```
git push
```

### Шаг 2 — Создай тег версии

```
git tag v1.0.0
git push origin v1.0.0
```

### Шаг 3 — Создай Release на сайте

1. Открой репозиторий на github.com
2. Справа нажми **"Releases"** → **"Create a new release"**
3. **Choose a tag** → выбери `v1.0.0`
4. **Release title:** `SecureTunnel v1.0.0`
5. В поле описания напиши что нового (см. шаблон ниже)
6. В секции **"Attach binaries"** → перетащи файл `dist\SecureTunnel.exe`
7. Нажми **"Publish release"**

### Шаблон описания Release:

```
## Скачать
→ SecureTunnel.exe — не требует Python, Windows 10/11 x64

## Установка
1. Скачай SecureTunnel.exe
2. Положи в отдельную папку
3. Запусти → нажми Start

## Что включено
- 3-узловой onion-маршрут: Entry → Middle → Exit
- TLS-in-TLS транспорт с Chrome-like fingerprint
- Постквантовый гибрид X25519 + ML-KEM-768
- SOCKS5 прокси :1080 и HTTP прокси :1081
- Автонастройка системного прокси Windows
- Поддержка 3 VPS в разных странах
```

### Прямая ссылка на скачивание (поделись ею):

```
https://github.com/ТВО_ИМЯ/secure_tunnel/releases/latest/download/SecureTunnel.exe
```

## Часть 7 — Следующие версии

Когда выпускаешь новую версию:

```
git add .
git commit -m "v1.1.0: описание изменений"
git push
git tag v1.1.0
git push origin v1.1.0
```

Затем снова создай Release на GitHub и прикрепи новый `.exe`.  
Ссылка `.../releases/latest/download/SecureTunnel.exe` автоматически будет вести на новую версию.
