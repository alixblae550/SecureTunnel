# SecureTunnel — Документация

---

> **Правовой статус:** SecureTunnel является учебно-демонстрационным программным проектом, разработанным в образовательных целях для изучения сетевых протоколов, криптографии и архитектуры защищённых соединений. Весь исходный код открыт и предназначен исключительно для законного использования в рамках действующего законодательства страны пользователя. Автор не несёт ответственности за использование программы в целях, нарушающих местное законодательство.

---

## 1. Что такое SecureTunnel

SecureTunnel — полностью рабочее приложение-туннель с графическим интерфейсом, реализующее **3-узловую луковую маршрутизацию** с многоуровневым шифрованием. Один `.exe` файл — никакой установки Python, никаких зависимостей.

Проект создан как учебная работа по углублённому изучению современных сетевых технологий: TLS, SOCKS5, HTTP CONNECT, луковой маршрутизации и асинхронного программирования на Python.

## 2. Архитектура системы

```
[Браузер / Telegram / Steam / любое приложение]
         │
         ▼
[HTTP CONNECT Proxy  :8080]  ← системный прокси Windows
         │
         ▼
[SOCKS5 Proxy  :1080]  ← поддержка UDP ASSOCIATE (RFC 1928)
         │
         ▼
[Tunnel Relay — пул соединений + ротация цепочки + cover-трафик]
         │  TLS-in-TLS + Hybrid KEM K1 (X25519 + ML-KEM-768)
         ▼
[Entry Node  :8765]  ← знает только relay и middle
         │  TLS-in-TLS + Hybrid KEM K2 (X25519 + ML-KEM-768)
         ▼
[Middle Node  :8766]  ← знает только entry и exit
         │  TLS-in-TLS + Hybrid KEM K3 (X25519 + ML-KEM-768)
         ▼
[Exit Node  :8767]  ← DoH-резолвинг, TCP+UDP, выход в интернет
         │
         ▼
[Интернет]
```

### Свойства безопасности цепочки

| Узел | Что знает | Что НЕ знает |
|------|-----------|--------------|
| Entry | IP relay, ключ K1, адрес middle | Назначение, payload |
| Middle | Адреса entry и exit, ключи K2 и K3 | IP relay, назначение |
| Exit | Реальный хост:порт назначения, ключ K3 | IP relay, IP entry |

### Уровни защиты

| Уровень | Протокол | Назначение |
|---------|----------|------------|
| 1 | Внешний TLS (cover SNI) | Маскировка под легитимный HTTPS-трафик |
| 2 | Внутренний TLS (MemoryBIO) | Второй слой шифрования поверх первого |
| 3 | X25519 ECDH × 3 | Три независимых сессионных ключа K1/K2/K3 |
| 4 | ML-KEM-768 × 3 | Постквантовая защита (при наличии OpenSSL ≥ 3.5) |
| 5 | AES-256-GCM | Шифрование полезной нагрузки |
| 6 | DNS over HTTPS (DoH) | Зашифрованный DNS через Cloudflare/Google |
| 7 | Bucket padding | Фиксированные размеры фреймов — анализ трафика бесполезен |

## 3. Компоненты

### 3.1 Графический лаунчер (`launcher.py`)

- Запуск/остановка всех узлов одной кнопкой
- Цепочка запуска по сигналам готовности (без фиксированных задержек)
- Автоматическое включение/выключение системного прокси Windows
- Авто-перезапуск упавших компонентов через 2 секунды
- Индикатор скорости (↓ download / ↑ upload) в реальном времени
- Kill Switch — кнопка блокировки всего трафика кроме туннеля
- Настройки портов через GUI (`settings.json`)
- Split tunneling — PAC файл с доменами которые идут напрямую
- Системный трей — сворачивается, не мешает работе
- Уникальный AUTH_SECRET генерируется при первом запуске (`auth_secret.key`)
- Цветной лог с буферизацией (до 2000 строк)

**Цепочка запуска:**
```
exit_node  → (сигнал) → middle node (node1)
node1      → (сигнал) → entry_node
entry_node → (сигнал) → socks5_proxy
socks5     → (пул готов) → http_proxy
http_proxy → (сигнал) → кнопки активны
```

### 3.2 Exit Node (`exit_node.py`)

- Единственный узел с доступом в интернет
- DoH-резолвинг: IPv4 (A) → IPv6 (AAAA) fallback
- Поддержка IPv6 соединений с целевыми серверами
- Поддержка UDP (одиночные датаграммы с таймаутом)
- Connection reuse — одно туннельное соединение для множества сессий

### 3.3 Middle Node / Entry Node (`node1.py`, `entry_node.py`)

- TLS-in-TLS туннели между узлами
- Гибридный ECDH handshake (X25519 + ML-KEM)
- Anti-probing — соединения без правильного токена отклоняются

### 3.4 Tunnel Relay (`tunnel_relay.py`)

- Пул соединений к entry node
- Ротация цепочки
- Синусоидальный cover-трафик (маскировка паттернов)
- Счётчики пропускной способности (↓/↑ KB/s каждую секунду)

### 3.5 SOCKS5 / HTTP прокси

- SOCKS5 (порт 1080): TCP + UDP ASSOCIATE
- HTTP CONNECT (порт 8080): для браузеров и системного прокси
- `reuse_address=True` — нет задержки при перезапуске

### 3.6 DoH Resolver (`doh_resolver.py`)

- Провайдеры: Cloudflare (`cloudflare-dns.com`) и Google (`dns.google`)
- Кэш с TTL, in-flight дедупликация (N параллельных запросов = 1 DoH запрос)
- IPv4 first, IPv6 fallback
- Пропускает DoH если hostname уже является IP адресом

## 4. Безопасность

### AUTH_SECRET
При первом запуске генерируется уникальный 32-байтный секрет (`auth_secret.key`) рядом с `.exe`. Он передаётся всем узлам через переменную окружения — без него соединения отклоняются. Не удаляй этот файл.

### Kill Switch
При активации добавляет правила Windows Firewall:
- Блокирует весь исходящий трафик
- Разрешает только процессу SecureTunnel

При остановке приложения правила автоматически удаляются.

### Anti-probing
Узлы не отвечают на соединения без правильного заголовка — сканеры не могут определить что на порту работает туннель.

## 5. Установка и запуск

### Требования
- Windows 10/11 (x64)
- Ничего больше — всё включено в `.exe`

### Запуск
1. Скачай `SecureTunnel.exe` из [Releases](../../releases)
2. Запусти `.exe`
3. Нажми **Start**
4. Настрой браузер на SOCKS5 прокси `127.0.0.1:1080` или HTTP прокси `127.0.0.1:8080`

Либо включи **системный прокси Windows** — лаунчер сделает это автоматически.

### Настройка браузера (Firefox)
```
Настройки → Сеть → Настройки соединения → Ручная настройка прокси
SOCKS-хост: 127.0.0.1   Порт: 1080   SOCKS v5
```

### Настройка браузера (Chrome)
```
chrome --proxy-server="socks5://127.0.0.1:1080"
```

## 6. Настройки

### Порты (через кнопку Settings в лаунчере)

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| SOCKS5 порт | 1080 | SOCKS5 прокси |
| HTTP порт | 8080 | HTTP/HTTPS прокси |

Настройки сохраняются в `settings.json` рядом с `.exe`.

### Split Tunneling
Кнопка **Split Tunneling** открывает диалог, где можно указать домены которые идут напрямую (без туннеля). Генерируется PAC файл, который автоматически применяется как системный прокси.

## 7. Для разработчиков

### Структура проекта

```
secure_tunnel/
├── launcher.py              # GUI лаунчер
├── secure_tunnel/
│   ├── config.py            # конфигурация узлов
│   ├── crypto.py            # X25519 + ML-KEM + AES-256-GCM
│   ├── framing.py           # фреймы с padding
│   ├── protocol.py          # MSG_DATA / MSG_COVER
│   ├── keyring.py           # X25519 ключи узлов
│   ├── anti_probing.py      # защита от сканирования
│   ├── circuit.py           # управление цепочкой
│   ├── doh_resolver.py      # DNS over HTTPS
│   ├── exit_node.py         # выходной узел
│   ├── node1.py             # middle узел
│   ├── entry_node.py        # входной узел
│   ├── tunnel_relay.py      # relay + cover-трафик
│   ├── socks5_proxy.py      # SOCKS5 прокси
│   ├── http_proxy.py        # HTTP CONNECT прокси
│   ├── transport/
│   │   └── tls_in_tls_transport.py
│   └── logging/
│       └── anon_logger.py
└── installer/
    └── SecureTunnel.spec    # PyInstaller сборка
```

### Сборка .exe

```bash
pip install pyinstaller pillow pystray cryptography msgpack keyring
pyinstaller installer/SecureTunnel.spec
```

Результат: `dist/SecureTunnel.exe`

### Развёртывание на серверах

Для полноценной анонимизации запусти каждый узел на отдельном VPS:

1. Скопируй репозиторий на три сервера
2. Измени `config.py` — укажи реальные IP адреса серверов
3. На сервере 1: `python -m secure_tunnel.exit_node`
4. На сервере 2: `python -m secure_tunnel.node1`
5. На сервере 3: `python -m secure_tunnel.entry_node`
6. Локально: запусти лаунчер
