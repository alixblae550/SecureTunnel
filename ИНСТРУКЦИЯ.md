## 1. Способ 1 — Готовый .exe (рекомендуется)

Не требует Python, pip или любых других инструментов.

### Шаг 1. Скачай `SecureTunnel.exe`

Файл находится в папке `dist/SecureTunnel.exe` проекта (18 МБ).

### Шаг 2. Помести в отдельную папку

```
C:\SecureTunnel\SecureTunnel.exe
```

> Важно: положи exe в **отдельную пустую папку** — при первом запуске рядом создадутся `cert.pem` и `key.pem`.

### Шаг 3. Запусти

Двойной клик на `SecureTunnel.exe` → откроется графический интерфейс → нажми **Start**.

**Готово.** Никаких дополнительных шагов не требуется.

## 2. Способ 2 — Из исходного кода

Подходит для разработки и кастомизации.

### 2.1 Требования

| Компонент | Минимум | Где скачать |
|-----------|---------|-------------|
| Windows 10/11 | 64-bit | — |
| Python | 3.11+ (лучше 3.12) | https://python.org/downloads |
| RAM | 512 МБ свободно | — |

### 2.2 Установка Python

1. Зайди на https://python.org/downloads
2. Скачай **Python 3.12** (Windows installer 64-bit)
3. Запусти установщик
4. **ВАЖНО:** поставь галочку **"Add Python to PATH"** перед нажатием Install
5. Нажми **Install Now**

Проверка (`Win+R → cmd`):
```
python --version
```
Должно показать: `Python 3.12.x`

### 2.3 Установка зависимостей

```
cd C:\путь\до\secure_tunnel
pip install cryptography msgpack keyring
```

### 2.4 Запуск

```
python launcher.py
```

### 2.5 Альтернатива — PowerShell-установщик

Правой кнопкой на `installer\install.ps1` → **"Запустить с PowerShell"**

Скрипт сам установит Python, зависимости, создаст ярлык на рабочем столе и предложит добавить в автозапуск.

## 3. Боевой режим — 3 VPS в разных странах

В режиме тестирования все узлы работают на одном компьютере — это удобно для проверки, но не даёт реальной анонимности. Для полноценной работы нужны 3 VPS-сервера.

```
[Твой ПК]  →  [VPS Entry]  →  [VPS Middle]  →  [VPS Exit]  →  Интернет
               Германия         Нидерланды        Финляндия
```

---

### 3.1 Что нужно купить

**3 VPS-сервера** с минимальными характеристиками:

| Параметр | Значение |
|----------|----------|
| ОС | Ubuntu 22.04 LTS |
| CPU | 1 vCPU |
| RAM | 512 МБ |
| Диск | 10 ГБ |
| Порт | 443 (TCP) открыт |
| Цена | ~€3–5/мес каждый |

**Рекомендуемые провайдеры:** Hetzner, Contabo, DigitalOcean, Vultr

> Выбери серверы в разных странах и у разных провайдеров.

---

### 3.2 Подключение к VPS

- **Windows 10/11:** встроенный OpenSSH (уже есть)
- **Альтернатива:** [PuTTY](https://putty.org) — бесплатный GUI-клиент

```
ssh root@IP_АДРЕС_VPS
```

---

### 3.3 Генерация общего секрета (один раз)

Секрет должен быть **одинаковым** на всех 3 VPS и в `.env` клиента.

```
python -c "import secrets; print(secrets.token_hex(32))"
```

Получишь строку вида:
```
a3f8c2d1e4b56f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1
```

**Сохрани её** — понадобится во всех следующих шагах.

---

### 3.4 Деплой Exit Node (VPS 3 — выходной)

> Exit Node — сервер, чей IP будут видеть сайты. Выбери страну исходя из нужного выходного IP.

**Скопируй проект на сервер (с ПК):**
```
scp -r C:\путь\до\secure_tunnel root@IP_VPS3:/tmp/securetunnel
```

**На VPS 3:**
```bash
cd /tmp/securetunnel
chmod +x deploy.sh
./deploy.sh --node exit --secret "ВАШ_СЕКРЕТ"
```

**Дождись вывода:**
```
════════════════════════════════════════
  SecureTunnel deploy complete
  Node:        exit
  Public IP:   X.X.X.X     ← запомни
════════════════════════════════════════
```

---

### 3.5 Деплой Middle Node (VPS 2)

**Скопируй проект (с ПК):**
```
scp -r C:\путь\до\secure_tunnel root@IP_VPS2:/tmp/securetunnel
```

**На VPS 2:**
```bash
ssh root@IP_VPS2
cd /tmp/securetunnel
chmod +x deploy.sh
./deploy.sh \
  --node middle \
  --secret "ВАШ_СЕКРЕТ" \
  --exit-host IP_VPS3
```

Запомни Public IP этого сервера.

---

### 3.6 Деплой Entry Node (VPS 1)

**Скопируй проект (с ПК):**
```
scp -r C:\путь\до\secure_tunnel root@IP_VPS1:/tmp/securetunnel
```

**На VPS 1:**
```bash
ssh root@IP_VPS1
cd /tmp/securetunnel
chmod +x deploy.sh
./deploy.sh \
  --node entry \
  --secret "ВАШ_СЕКРЕТ" \
  --middle-host IP_VPS2 \
  --exit-host IP_VPS3
```

---

### 3.7 Настройка клиента

Создай файл `.env` рядом с `SecureTunnel.exe` (или в папке проекта):

```
ENTRY_HOST=IP_VPS1
ENTRY_PORT=8765
MIDDLE_HOST=IP_VPS2
MIDDLE_PORT=8766
EXIT_HOST=IP_VPS3
EXIT_PORT=8767
AUTH_SECRET=ВАШ_СЕКРЕТ
```

`.env` читается автоматически при запуске — никакие другие настройки не нужны.

---

### 3.8 Проверка

Открой браузер → зайди на https://2ip.ru

Должен отображаться **IP Exit Node**, а не твой домашний IP.

## 4. Настройка браузера и приложений

После нажатия Start лаунчер автоматически настраивает **системный прокси Windows** — большинство приложений заработают без дополнительных настроек.

### 4.1 Что работает автоматически

| Приложение | Метод |
|-----------|-------|
| Telegram Desktop | Системный прокси |
| Microsoft Edge | Системный прокси |
| Discord | Системный прокси |
| Steam | Системный прокси |
| Любые .NET / WinINet приложения | Системный прокси |

### 4.2 Google Chrome

В лаунчере есть кнопка **"Open Chrome"** — запускает Chrome через SOCKS5 автоматически.

Или вручную:
```
"C:\Program Files\Google\Chrome\Application\chrome.exe" --proxy-server="socks5://127.0.0.1:1080"
```

### 4.3 Firefox

1. Настройки → Основные → Параметры сети → Настроить
2. **Ручная настройка прокси**
3. SOCKS хост: `127.0.0.1`, Порт: `1080`, SOCKS v5
4. Галочка **"Использовать DNS через SOCKS5"**
5. OK

### 4.4 Любое ПО с поддержкой HTTP-прокси

```
Прокси: 127.0.0.1
Порт:   1081
```

### 4.5 Проверка через curl

```
curl --socks5 127.0.0.1:1080 https://httpbin.org/ip
```

Должен вернуть IP Exit Node.

## 5. Управление и мониторинг

### 5.1 Интерфейс лаунчера

```
┌──────────────────────────────────────────────────┐
│  SecureTunnel                                    │
│                                                  │
│  [Start]  [Stop]  [Open Chrome]                  │
│                                                  │
│  Relay pool:  ready ●                            │
│  Entry pool:  20/20 ●                            │
│  Middle pool: 20/20 ●                            │
│                                                  │
│  [exit] listening on 127.0.0.1:8767              │
│  [node1] exit pool: 20/20 ready                  │
│  ...                                [Copy Log]   │
└──────────────────────────────────────────────────┘
```

| Элемент | Описание |
|---------|----------|
| Start | Запустить всю цепочку |
| Stop | Остановить всё, отключить системный прокси |
| Open Chrome | Запустить Chrome через SOCKS5 |
| Relay pool: ready | Туннель к Entry Node готов |
| Entry pool: 20/20 | Пул Entry→Middle |
| Middle pool: 20/20 | Пул Middle→Exit |
| Copy Log | Скопировать лог в буфер обмена |

### 5.2 Автоперезапуск

- **На ПК:** упавший компонент автоматически перезапускается через 2 секунды
- **На VPS:** systemd перезапускает сервис через 3 секунды

### 5.3 Мониторинг VPS

```bash
# Статус
systemctl status securetunnel-exit

# Живой лог
journalctl -u securetunnel-exit -f

# Перезапуск
systemctl restart securetunnel-exit
```

Замени `exit` на `middle` или `entry` для других узлов.

## 6. Обновление на VPS

**С ПК — скопируй новую версию:**
```
scp -r C:\путь\до\secure_tunnel root@IP_VPS1:/tmp/st_new
scp -r C:\путь\до\secure_tunnel root@IP_VPS2:/tmp/st_new
scp -r C:\путь\до\secure_tunnel root@IP_VPS3:/tmp/st_new
```

**На каждом VPS:**
```bash
systemctl stop securetunnel-NODE

# Сохранить сертификат
cp /opt/securetunnel/cert.pem /tmp/cert.bak
cp /opt/securetunnel/key.pem  /tmp/key.bak

# Обновить файлы
cp -r /tmp/st_new/* /opt/securetunnel/
cp /tmp/cert.bak /opt/securetunnel/cert.pem
cp /tmp/key.bak  /opt/securetunnel/key.pem

# Обновить зависимости
/opt/securetunnel/.venv/bin/pip install -q --upgrade cryptography msgpack

systemctl start securetunnel-NODE
```

## 7. Решение проблем

### `OSError: [Errno 10048] address already in use`

Порт занят предыдущей сессией. Лаунчер убивает старые процессы автоматически при старте. Если ошибка повторяется:

```
netstat -ano | findstr :8767
taskkill /F /PID <PID>
```

---

### `Server disconnected during inner TLS handshake`

Rate limiter блокировал соединения с localhost — исправлено в текущей версии. Проверка:
```
python -c "from secure_tunnel.anti_probing import check_rate; print(check_rate('127.0.0.1'))"
```
Должно вернуть `True`.

---

### `UserWarning: AUTH_SECRET is set to the default value`

Предупреждение, не ошибка. Для локального тестирования не критично. Для VPS — обязательно задай свой секрет в `.env`.

---

### `UserWarning: ML-KEM-768 unavailable`

Постквантовое шифрование недоступно. Система работает на X25519 — по-прежнему безопасно. В `.exe` это предупреждение не влияет на работу.

---

### Пул долго заполняется или остаётся пустым

1. Убедись что все три узла показывают статус listening в логе
2. Нажми Stop → Start
3. Проверь интернет-соединение

---

### Сайты не открываются при статусе "All systems ready"

1. Проверь системный прокси: Настройки Windows → Сеть → Прокси → должен быть `127.0.0.1:1081`
2. Проверь через curl: `curl -x http://127.0.0.1:1081 https://httpbin.org/ip`
3. Используй кнопку **"Open Chrome"** — она гарантированно идёт через туннель

---

### На VPS: сервис не запускается

```bash
journalctl -u securetunnel-exit -n 50 --no-pager

# Порт занят?
ss -tlnp | grep 8767

# Сертификат на месте?
ls /opt/securetunnel/cert.pem /opt/securetunnel/key.pem

# Секрет верный?
cat /etc/securetunnel/env
```
